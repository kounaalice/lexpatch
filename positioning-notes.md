# LexCard ポジショニング

## サービス概要（lexcard.jp）

| 項目 | 内容 |
|------|------|
| コア機能 | e-Gov法令API準拠の法令検索・条文閲覧（9,000+法令） |
| 独自機能 | 改正提案、法令プロジェクト、逐条解説、コミュニティ |
| データソース | デジタル庁 e-Gov法令検索（公開データ・商用利用可） |
| 判例 | 外部リンクのみ（データ保持なし） |
| 特徴 | ダークモード、英語版対応 |

## 決定した立ち位置

### 基本構造：営利企業が無料サービスを運営し、社会的信用で別事業につなげる

```
営利企業（合同会社 or 株式会社）が運営
  │
  ├─ LexCard（完全無料・課金なし）
  │    ├─ 法令検索・閲覧
  │    ├─ 改正提案・コミュニティ
  │    ├─ 逐条解説
  │    └─ 判例は外部リンクのみ
  │
  ├─ コード・システムは企業資産として厳格に管理
  │
  └─ 収益化
       ├─ カスタマイズ別注（コード・システムの移植）
       └─ 社会的信用を活かした別事業
```

### サービスは無料、システムは売り物

LexCard自体では課金しない。ただし営利企業運営であるため、
コード・システムの権利は企業に帰属し、厳格に管理する。

### 利用規約上の線引き

**無料で使える（利用者として）**
- 法令検索・閲覧
- 改正提案・コミュニティ参加
- 逐条解説の閲覧

**禁止（無断では）**
- ソースコード・システムの複製・転用
- APIの無断大量利用・再販
- LexCardのデータ・UIを使った類似サービスの構築

**有償で対応（別注）**
- コード・システムを移植してのカスタマイズ
- 自治体/企業向けの専用環境構築
- 特定用途向けのAPI提供

## 収益化の方向性

### LexCard自体では稼がない理由

- 課金機能・決済・請求・サポート対応が不要 → 一人運営の負担を大幅軽減
- 「完全無料」は強いブランド。公共データを扱うサービスとして自然
- 有料の壁がないので利用者が増えやすい

### 収益源

1. **カスタマイズ別注** — LexCardのコード・システムを移植しての専用環境構築（自治体・企業・団体向け）
2. **社会的信用を活かした別事業** — 法令アクセスを改善する公共的サービスの運営実績を基盤に展開
   - リーガルテック系の受託開発・コンサル
   - 自治体・省庁向けDX支援
   - 法令データ活用に関する講演・執筆
   - 別のSaaSプロダクト（LexCardの実績がポートフォリオ）

## 営利企業運営であることの意義

| 非営利だった場合 | 営利企業だからこそ |
|---|---|
| コードの公開を求められがち | コード・システムは企業資産として保護 |
| 「無料なんだから共有すべき」という圧力 | 「無料で使えるが、中身は売り物」と明確 |
| カスタマイズ受注がしにくい | 別注として自然に受けられる |
| 法人を分ける管理コスト | 一法人で完結（一人運営に適合） |

## 検討経緯

以下の選択肢を検討した上で、現在の方針に至った。

### 棄却案A: 非営利型（営利は別事業として分離）
- 一人で二法人の運営は現実的でないため棄却

### 棄却案B: 営利型（リーガルテックSaaSとして既存有料DB系と競合）
- Westlaw Japan、LegalForce、LEX/DB等とデータ量・営業力・開発リソースで勝てないため棄却
- 判例データを扱わない方針により、さらに競合の土俵から離れる

### 棄却案C: ハイブリッド型（非営利法人＋営利法人）
- 棄却案Aと同じく、一人で二法人は非現実的

### 棄却案D: LexCard内でのフリーミアム課金
- 課金関連の運用コストが一人運営に合わない
- 「貰わない方が手間取られなくて楽」

## データソースに関する整理

| データソース | 商用利用 | 学術目的のみ | LexCardでの扱い |
|---|---|---|---|
| e-Gov法令API | 可（出典記載のみ） | 制限なし | メインデータソース |
| NII 判例HTMLデータ | 不可 | 学術研究限定 | 使用しない |
| 裁判所 判例検索 | APIなし | — | 外部リンクのみ |

判例をリンクのみとすることで、学術目的限定APIの制約を回避。
e-Gov法令APIは商用利用可のため、営利企業運営でも問題なし。
