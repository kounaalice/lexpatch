# LexCard ポジショニング検討メモ

## 現状整理（lexcard.jp）

| 項目 | 内容 |
|------|------|
| コア機能 | e-Gov法令API準拠の法令検索・条文閲覧（9,000+法令） |
| 独自機能 | 改正提案、法令プロジェクト、逐条解説、コミュニティ |
| 料金 | 無料・登録不要（組織向けプランあり） |
| データソース | デジタル庁 e-Gov法令検索（公開データ） |
| 特徴 | ダークモード、英語版対応 |

## 検討した立ち位置の選択肢

### A. 非営利型（営利は別事業として分離）

- 法令情報を公共財として提供
- 改正提案・コミュニティ機能との親和性が高い
- 助成金・自治体連携がしやすい
- 参考: Wikipedia、OpenStreetMap、Code for Japan

### B. 営利型維持（リーガルテックSaaS）

- 法律事務所・企業法務向け生産性ツールとして収益化
- 競合: Westlaw Japan、LegalForce、LAWGUE等
- 組織プラン月額課金、API利用料で収益化

### C. ハイブリッド型

```
┌─────────────────────────────────────────┐
│  営利レイヤー（別事業 or 上位プラン）      │
│  ・法人向けAPI  ・AI法令分析              │
│  ・改正影響評価  ・コンプライアンス管理    │
├─────────────────────────────────────────┤
│  公共レイヤー（LexCard本体）              │
│  ・法令検索/閲覧  ・改正提案              │
│  ・逐条解説  ・コミュニティ               │
│  → 無料・登録不要を維持                   │
└─────────────────────────────────────────┘
```

## 方向性の暫定結論

LexCardの独自性は「改正提案・コミュニティ機能」にある。単なる法令検索ならe-Govで十分だが、
「市民が法令改正に参加できるプラットフォーム」は他にない価値。

**ハイブリッド型（C）を推奨検討案とする：**

1. 公共レイヤーを非営利的に運営 → 改正提案への参加者・コミュニティの信頼性向上
2. 営利レイヤーで法人向け機能を別事業展開（AI法令分析、コンプライアンス管理等）
3. 「幅広く普及」と「別事業での収益化」の両立が可能

## ユーザー回答（検討時の前提）

- ターゲット: 幅広く普及
- 差別化: 名刺に限らない広いプラットフォーム
- 収益源: サブスクリプション + 別事業での収益化（一部未定）
